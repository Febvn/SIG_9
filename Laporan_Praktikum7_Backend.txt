2. PERSIAPAN DAN INSTALASI
Library yang digunakan dalam praktikum ini antara lain:
FastAPI: Framework utama untuk pembuatan API.
Uvicorn: Server ASGI untuk menjalankan FastAPI.
Asyncpg: Driver database PostgreSQL yang mendukung operasi asynchronous.
Pydantic: Digunakan untuk validasi skema data input.
Python-dotenv: Untuk mengelola variabel lingkungan (koneksi database).

3. IMPLEMENTASI KODE
3.1 Skema Validasi (Pydantic)
Sesuai ketentuan, digunakan model Pydantic untuk memvalidasi input saat melakukan penambahan data (POST).
(Sertakan screenshot kode model FasilitasCreate)

3.2 Endpoint API
Telah berhasil diimplementasikan 5 endpoint utama:
GET /fasilitas/: Mengambil seluruh daftar fasilitas publik.
GET /fasilitas/{id}: Mengambil detail fasilitas berdasarkan ID unik.
GET /fasilitas/geojson: Menghasilkan output data dalam format FeatureCollection GeoJSON agar dapat dikonsumsi oleh aplikasi frontend seperti Leaflet.
POST /fasilitas/: Menambahkan titik fasilitas baru ke database menggunakan fungsi ST_Point dan ST_SetSRID.
GET /fasilitas/nearby: Melakukan query spasial menggunakan fungsi ST_DWithin untuk mencari fasilitas dalam radius tertentu.

5. ANALISIS
Integrasi Spasial: Penggunaan ST_AsGeoJSON pada database sangat memudahkan backend karena format data sudah siap digunakan oleh standar GIS tanpa perlu konversi manual di sisi aplikasi.
Validasi Data: Pydantic mencegah terjadinya error pada database jika user memasukkan tipe data yang salah atau koordinat yang tidak masuk akal.
Kinerja: Penggunaan asyncpg memungkinkan API menangani banyak permintaan secara bersamaan (concurrency) dengan performa yang lebih baik dibandingkan driver sinkron standar.
